HTML Structure

  • Guess what. There are a bunch of people (World Wide Web Consortium (W3C)) who invented this thing called HTML.
  • Most page parts we just discussed actually map quite nicely on what we call HTML tags or elements.
    • Article => article
    • Paragraph => p
    • Headings => h1, h2, h3, ...
    • Lists => ul, ol and li
    • Image => img and figure

HTML is a markup language

  • Structural markup
    • Elements (tags) are used to describe the structure of the document.
  • Semantic markup provides extra information, such as
    • where emphasis is placed in a sentence
    • meaning of acronyms
    • that something you have written is a quotation (and who said it)

Structural markup

These elements allow us to define the structure of a page. 


<h1>Hello world</h1>
  • We know "Hello World" is a heading of level 1.
  • Now the browser knows too.
  • Firefox may choose to display this bold.
  • Internet Explorer browser may choose to display it bold and underlined.
  • As long as we do not specify how it should be represented (we use CSS for that) the browser is free to choose how to display this to the user.

Semantic markup

These elements are not intended to affect the structure of your web pages, but they do add extra information to the pages. 


<blockquote cite="http://en.wikipedia.org/wiki/ Winnie-the-Pooh">
  <p>Did you ever stop to think, and forget to start again?</p>
</blockquote>
    Blockquote element:
  • Used for longer quotes that take up an entire paragraph
  • Browsers tend to indent the contents of the element
  • However you should not use this element just to indent a piece of text

The olden days

In the olden days HTML was also a presentational markup language. It allowed you to specify how data had to be presented.


<font face="verdana" size="3" color="red">
This is some text!
</font>

We have however evolved and seen the error of our ways.

Now we use CSS(3) for our design needs (but more on that later).

Doctype


<!DOCTYPE html>
  • Required in HTML5 documents so that browsers render the page in standard mode.
  • Some browsers operate in quirks mode to maintain backward compatibility with web pages that are not up-to-date with the latest standards.

Html element

  • The html element encloses the head and body section.

<html>
    <head>
        ...
    </head>
    <body>
        ...
    </body>
</html>

The head section

Metatags

The head section contains information about the HTML5 document.


<head>
    <!-- Character encoding for the HTML document -->
    <meta charset = "utf-8">
    <title>Hello World</title>
</head>
  • Metatags
    • Provide metadata about the HTML document
    • Mostly name / value pairs
    • Ex. The character set that the page uses
      • UTF-8 is most popular character-encoding scheme
      • Helps the browser determine how to render the content

Alot of metatags exist.

The head section

Title


<head>
    <!-- Character encoding for the HTML document -->
    <meta charset = "utf-8">
    <title>Hello World</title>
</head>
  • The title of the page
    • Describes the web page.
    • Usually appears in the title bar at the top of the browser window.
    • Or in the browser tab on which the page is displayed.
    • As the text identifying a page when users add the page to their list of Favorites (Bookmarks).
    • Search engines use the title for indexing purposes and when displaying results.

The head section

CSS and JS

  • Can also contain document-formatting instructions (ex. CSS3) and client-side scripts for creating dynamic web pages (ex. JavaScript).

<head>
    ...
    <style type="text/css">
        ...
    </style>
    <script type="text/javascript">
        ...
    </script>
</head>

The body section


<body>
    <p>Hello World</p>
</body>
  • Contains the page's content, which the browser displays when the user visits the web page.
  • Contains text (and images, animations, audio and videos) that represents the content of a document and elements that specify a document's structure and meaning.

Comments


<!-- Character encoding for the HTML document -->
  • Improve readability.
  • Can be inserted throughout your html files.
  • Ignored by browser upon rendering.
  • Not directly visible to user but still there (viewable through source)

HTML tags

Did anyone notice the html tags ?

HTML tags

    HTML5 documents delimit most elements with a start tag and end tag.
  • A start tag consists of the element name in angle brackets.
  • An end tag consists of the element name preceded by a forward slash (/) in angle brackets.

HTML tags

Void Elements

    There are several so-called "void elements" that do not have end tags.
  • Examples:
    • Line break: <br>
    • Horizontal line: <hr>

HTML attributes

Many start tags have attributes that provide additional information about an element, which browsers use to determine how to process the element.

  • Each attribute has a name and a value separated by an equals sign (=).
  • An end tag consists of the element name preceded by a forward slash (/) in angle brackets.
  • Examples: 
    
<meta charset="utf-8">
<p class="css_class">Some text</p>
<img height="200px" src="800px-PSP-Homebrew.jpeg">

Whitespace in HTML

  • Most whitespace is ignored.
  • Line breaks (new lines) are converted to a space.
  • So in other words use whitespace to make your code readable.

Its all about them standards

Standard overview

HTML

  • Presentational and structural markup
    • HTML 2.0 (1995)
    • HTML 3.2 (1997)
    • HTML 4.01 (1999)
  • Move towards structural markup
    • HTML 4.01 Transitional: allows for hybrid markup
    • HTML 4.01 Strict: seperate presentation
    • HTML 4.01 Frameset: body is replaced by frameset

Standard overview

XHTML

  • Stricter syntax: needs to be well-formed to allow parsing by XML parsers
    • XHTML 1.0 Transitional
    • XHTML 1.0 Strict
    • XHTML 1.0 Frameset

    ...
  </food>

  • Valid
  • Much simpler

Images

    The most popular image formats used by web developers today are
  • PNG (Portable Network Graphics)
  • JPEG (Joint Photographic Experts Group)
  • The src (source) attribute specifies an image's location relative to the document's location.

Building Blocks

Images


<img src="path/to/image.jpg" alt="description of image" >
  • The alt (alternative) attribute
    • Every <img> element should have an alt attribute, which contains text that is displayed if the client cannot render the image.
    • The alt attribute is also important for accessibility. Speech synthesizer software can speak the alt attribute's value so that a visually impaired user can understand what the browser is displaying.
  • Other optional attributes include width, height and title.

Hyperlinks


<a href="http://www.google.com">Google</a>
<a href="http://www.facebook.com">Facebook</a>
  • A hyperlink references or links to other resources, such as HTML5 documents and images.
  • Web browsers typically underline text hyperlinks and color them blue by default.
  • Hyperlinks are created using the <a> (anchor) element.

Building Blocks

Hyperlinks

  • Attribute href (hypertext reference) specifies a resource's location, such as
    • a web page or location within a web page (internal link) 
      <a href="#elementid">Click to go to id</a>
    • a file
    • an e-mail address 
      <a href="mailto:me@domain.com">Mail me</a>
  • If the web server cannot locate a requested document, it returns an error to the browser (known as a 404 error). The browser displays a page containing an error message.

Elements

Dit you notice something inherently different between <h1>, <a> and <p> tags ?

Some tags are rendered with a new line before and after and others are not.

Block and Inline elements

  • Block elements
    • appear on a single line
  • Inline elements
    • can appear in the middle of a line of text
  • Block elements may contain block and inline elements.
  • Inline elements may only contain inline elements (exception: <a> can also contain block elements)

Important! This will return when we see CSS.

Element overview

  • Headings
    • <h1>, <h2>, <h3>, ...
    • Block element
  • Paragraph
    • <p>...</p>
    • Block element
  • Image
    • <img src="...">
    • Block element
    • Void element

Element overview

  • Hyperlink
    • <a href="...">...</a>
    • Inline element
  • Unordered list
    • <ul><li>...</li></ul>
    • Block element
