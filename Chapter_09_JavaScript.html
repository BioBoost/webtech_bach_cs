Introduction

A scripting language

  • JavaScript is the most popular scripting language in the world. It is the standard language of the web, and widely used in servers, desktops, and mobile phones.
  • JavaScript is a Scripting Language
    • A scripting language is a lightweight programming language that supports the writing of scripts.
    • JavaScript was designed to add interactivity to HTML pages.
    • JavaScript is programming code that can be inserted into HTML pages to be executed by web browser.

Running JavaScript

  • All major web browsers contain JavaScript interpreters, which process the commands written in JavaScript.
  • Before you can run code examples with JavaScript on your computer, you may need to change your browser’s security settings.
    • IE9 prevents scripts on the local computer from running by default
    • Firefox, Chrome, Opera, Safari, ... browsers have JavaScript enabled by default.

Capabilities of JavaScript

  • JavaScript can write directly into the HTML output stream.
  • JavaScript can change the content of HTML elements.
  • JavaScript can react to events.
  • JavaScript can manipulate HTML styles.
  • JavaScript can test input, and alert about errors.
  • ...

JavaScript Logical Operators

  • JavaScript evaluates to false or true all expressions that include relational operators, equality operators and/or logical operators.
    • In JavaScript, any nonzero numeric value in a condition evaluates to true, and 0 evaluates to false.
    • Any string containing one or more characters evaluates to true, and the empty string "" evaluates to false.

Reference parameters

Reference parameters

  • There are two ways to pass arguments to functions (or methods)
    • Pass-by-value
      • a copy of the argument's value is made and is passed to the called function
    • Pass-by-reference
      • The caller gives the called function access to the caller’s data and allows the called function to modify the data if it so chooses.
      • Can improve performance because it can eliminate the overhead of copying large amounts of data, but it can weaken security because the called function can access the caller's data.

Reference parameters

  • In JavaScript, numbers, boolean values and strings are passed to functions by value.
  • All objects are passed to functions by reference.
  • With pass-by-value, changes to the copy of the value received by the called function do not effect the original variable's value in the calling function.
  • When information is returned from a function via a return statement, numbers and boolean values are returned by value and objects are returned by reference.

Passing arrays to functions

  • To pass an array as an argument to a function
    • Specify the array's name (a reference to the array) without brackets
    • Although entire arrays are passed by reference, individual numeric and boolean array elements are passed by value exactly as simple numeric and boolean variables are passed
      • Such simple single pieces of data are called scalars, or scalar quantities.
      • To pass an array element to a function, use the indexed name of the element as an argument in the function call.

Manipulating the Document Object Model

Document Object Model

  • When a web page is loaded, the browser creates a Document Object Model of the page.
  • The HTML DOM model is constructed as a tree of Objects

Document Object Model

  • The Document Object Model gives you scripting access to all the elements on a web page.
  • Using JavaScript, you can create, modify and remove elements in the page dynamically.
  • With the DOM, HTML5 elements can be treated as objects
    • and many attributes of HTML5 elements can be treated as properties of those objects.
  • Then objects can be scripted with JavaScript to achieve dynamic effects.

Document Object Model

  • The nodes in a document make up the page's DOM tree, which describes the relationships among elements.
    • Nodes are related to each other through child-parent relationships.
    • A node can have multiple children, but only one parent.
    • Nodes with the same parent node are referred to as siblings.
    • The html node in a DOM tree is called the root node, because it has no parent.

The Node Object

  • The Node object represents a node in the HTML document.
  • A node in an HTML document is:
    • The Document
    • An element
    • An attribute
    • Text
    • A comment

Event bubbling

  • The process whereby events fired on child elements bubble up to their parent elements.
  • When an event is fired on an element, it is first delivered to the element's event handler (if any), then to the parent element's event handler (if any), ...
  • If you intend to handle an event in a child element alone, you should cancel the bubbling of the event in the child element's event-handling code by using the cancelBubble property of the event object.

The JS code to register event handlers 

<script type="text/javascript">
function registerEvents() {
    document.addEventListener("click", documentClick);                        
    document.getElementById("bubble").addEventListener("click", bubble);    
    document.getElementById("noBubble").addEventListener("click", noBubble);    
}
window.addEventListener("load", registerEvents);
</script>

Event bubbling

  • Notice that the event object can be passed to the event handler.

<script type="text/javascript">
function documentClick() {
    alert( "You clicked in the document." );  
}

function bubble(e) {
    alert( "This will bubble." );
    e.cancelBubble = false;
}

function noBubble(e) {
    alert( "This will not bubble." );
    e.cancelBubble = true;
}
</script>

Legacy Code for event handling

  • Two other models for registering event handlers
    • Inline model
    • Traditional model
  • Both are considered legacy code by the HTML5 standard but are still frequently used.

Legacy Code - Inline model

  • Inline model treats events as attributes of HTML elements
    • Places calls to JavaScript functions directly in HTML code.
    • It's generally considered poor practice to intermix HTML and JavaScript code in this manner.

<body onload="loadEventHandler()">

Legacy Code - Traditional model

  • Traditional model assigns the name of the function to the event property of a DOM node (in JavaScript code).
    • Problem: If another statement assigns a value to the document.onload property, the original value is lost, which may not be the intended result.
    • Multiple events handlers can not be specified directly (can be achieved using anonymous functions or wrapper functions).

document.onload = "loadEventHandler()"
