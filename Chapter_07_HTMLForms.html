HTML forms

Usage

  • HTML5 provides forms for collecting information from users.
  • Examples
    • Registrations forms
    • Giving feedback
    • Search functionality
    • Posting comments

Building blocks

  • There are several types of form controls that you can use to collect information from visitors to your site.

Process flow

  1. A user fills in a form and then presses a button to submit the information to the server.

Process flow

  1. The name of each form control is sent to the server along with the value the user enters or selects.
  2. The server processes the information using a programming language such as PHP, C#, VB.net, or Java. It may also store the information in a database.
  3. The server creates a new page to send back to the browser based on the information received.

Processing forms on the server

  • A form may have several form controls, each gathering different information.
  • The server needs to know which piece of inputted data corresponds with which form element.
  • Name and id attributes
    • Not always done in example code to save space.
    • Name is used as identifier of input when a form is posted back to the server.
    • ID is used by JavaScript, CSS, internal links, ...
    • Best practice: set name and id attribute to same identifier.

Basic form construction

  • A form is defined by a form element

<form method="post" action="myformhandler.php" id="myfeedbackform">
    <!-- Form controls are placed here -->
</form>
  • Attribute method specifies how the form's data is sent to the web server.
    • We will come back to this later
  • The action attribute of the form element specifies the script to which the form data will be sent.
  • The id attribute is used to identify the form distinctly from other elements on the page

Populating forms with input controls

  • The <input> element is used to create several different form controls. The value of the type attribute determines what kind of input they will be creating.
  • Examples: hidden, text, password, checkbox, radio, submit, reset, ...

Text input

  • Used for a single line of text such as address or name.
  • The label element provides users with information about the input element's purpose.

<form method="post" action="myformhandler.php" id="myfeedbackform">
  <div>
    <label>
      Please state your name: 
      <input type="text" placeholder="firstname lastname"
            id="firstnamelastname" name="firstnamelastname" maxlength="25">
    </label>
  </div>
</form>
  • name identifies the form control
  • maxlength limits the number of characters a user may enter
  • placeholder shows a temporary text in the text field (not default value!).

Password input

  • When the type attribute has a value of password it creates a text box that acts just like a single-line text input, except the characters are blocked out.
  • Although the password is hidden on the screen, this does not mean that the data in a password control is sent securely to the server. You should never use these for sending sensitive data such as credit card numbers.

<form method="post" action="myformhandler.php" id="myfeedbackform">
  <div>
    <label>
      Please enter your password: 
      <input type="password" placeholder="password"
            id="password" name="password">      
    </label>
  </div>
</form>

Textarea

  • The textarea element is used to create a multi-line text input.
  • Unlike other input elements this is not an empty element. It should therefore have an opening and a closing tag.
  • Use CSS to control the width and height of a textarea element.
    • Older forms used the cols and rows attributes

<form method="post" action="myformhandler.php" id="myfeedbackform">
  <div>
    <label>
      Enter your comments here
      <textarea name="comments" id="comments">...</textarea>
    </label>
  </div>
</form>

Radio buttons

  • Radio buttons allow users to pick just one of a number of options.

<form method="post" action="myformhandler.php" id="myfeedbackform">
  <div>
    <label>
      Please select your favorite action hero:
      <input type="radio" value="rambo"
            id="hero_rambo" name="hero" checked="checked">Rambo

      <input type="radio" value="madmax"
            id="hero_mm" name="hero">Mad Max

      <input type="radio" value="bugsbunny"
            id="hero_bb" name="hero">Bugs Bunny
    </label>
  </div>
</form>

Radio buttons

  • name the value of the name attribute should be the same for all of the radio buttons used in a group
  • value indicates the value that is sent to the server for the selected option. The value of each of the buttons in a group should be different (so that the server knows which option the user has selected).
  • checked can be used to indicate which value (if any) should be selected when the page loads. The value of this attribute is checked. Only one radio button in a group should use this attribute.

Checkboxes

  • Checkboxes allow users to select (and unselect) one or more options in answer to a question.
  • Similar to radio buttons
  • Checked attribute can also be used
  • Names should be same and values should be different

<form method="post" action="myformhandler.php" id="myfeedbackform">
  <div>
    <label>
      What programming languages do you know:
      <input type="checkbox" value="C"
            id="prog_c" name="proglang">C

      <input type="checkbox" value="Cpp"
            id="prog_cpp" name="proglang">C++

      <input type="checkbox" value="Csharp"
            id="prog_csharp" name="proglang">CSharp
    </label>
  </div>
</form>

Drop down list box

  • A drop down list box (also known as a select box) allows users to select one option from a drop down list.
  • The select element is used to create a drop down list box. It contains two or more option elements.

<form method="post" action="myformhandler.php" id="myfeedbackform">
  <div>
    <label>
      Please select your favorite music:
      <select name="music" id="music">
        <option value="heavymetal"
            selected="selected">Heavy Metal</option>
        <option value="blackmetal"
            selected="selected">Black Metal</option>
        <option value="othermetal"
            selected="selected">Other Metal</option>
      </select>
    </label>
  </div>
</form>
  • selected can be used to indicate the option that should be selected when the page loads.

Submitting your form

  • When the submit button is pressed, the form's data is sent to the location specified in the form's action attribute.

<form method="post" action="myformhandler.php" id="myfeedbackform">
  <input type="submit" name="submit" value="Submit your comment" />
</form>
  • value controls the text that appears on the button.

HTTP GET and HTTP POST

Get and Post

  • The two most common HTTP request types (also known as request methods) are get and post.
    • A get request typically gets (or retrieves) information from a server, such as an HTML document, an image or Cascaded Style Sheet (CSS)
    • A post request typically posts (or sends) data to a server.
      • Common uses of post requests are to send form data or documents to a server.

Form data

  • An HTTP request often posts data to a server-side form handler that processes the data.
  • For example, when a user performs a search or participates in a web-based survey, the web server receives the information specified in the HTML form as part of the request.
  • Get requests and post requests can both be used to send data to a web server, but each request type sends the information differently.

Submitting form data using a GET request

  • A get request appends data to the URL, e.g., www.google.com/search?q=html5
    • q=html5 is called the query string
      • q is the name of a variable in Google’s search form and html5 is the search term.
      • A name/value pair is passed to the server separated by an equals sign (=).
      • If more than one name/value pair is submitted, each pair is separated by an ampersand (&).
    • search is the name of the server-side form handler
    • The ? separates the query string from the rest of the URL in a request.

Submitting form data using a GET request

  • The server uses data passed in a query string to retrieve an appropriate resource from the server.
  • The server then sends a response to the client.
  • A get request may be initiated by submitting an HTML form whose method attribute is set to "get", or by typing the URL (possibly containing a query string) directly into the browser's address bar.

Submitting form data using a POST request

  • A post request sends form data as part of the HTTP message, not as part of the URL.
  • A get request typically limits the query string (i.e., everything to the right of the ?) to a specific number of characters, so it’s often necessary to send large amounts of information using the post method.
  • The post method is also sometimes preferred because it hides the submitted data from the user by embedding it in an HTTP message.

Submitting form data using a POST request

  • If a form submits several hidden input values along with user-submitted data, the post method might generate a URL like www.searchengine.com/search
  • The form data still reaches the server and is processed in a similar fashion to a get request, but the user does not see the exact information sent.

Submitting form data using a POST request

  • The data send in a post request is not part of the URL, and the user can't see the data by default.
  • However, tools are available that expose this data, so you should not assume that the data is secure just because a post request is used.

Example of GET

  • Using method="get" appends the form data directly to the end of the URL of the script, where it's visible in the browser's Address field.

<form method="get" action="Forms_GET.php">
    <p>
        <label>Name:
            <input type="text" name="txtName" maxlength="30">
        </label>
        <input type="hidden" name="txtHidden" value="aHiddenValue">
    </p>
    <p>
        <input type="submit" value="Submit">
        <input type="reset" value="Clear">
    </p>   
</form>

Processing form data using PHP

Form submitted using GET

  • When the form is submitted using the GET method, then the data is available as an associative array via the superglobal $_GET

<?php    
    if (!empty($_GET['txtName']) && !empty($_GET['txtHidden'])) {
        echo "<p>Username: " . $_GET['txtName'] . "</p>";
        echo "<p>Hidden value: " . $_GET['txtHidden'] . "</p>";
    }
    else {
        echo "No values submitted";
    }
?>

Form submitted using POST

  • When the form is submitted using the POST method, then the data is available as an associative array via the superglobal $_POST

<?php    
    if (!empty($_POST['txtName']) && !empty($_POST['txtHidden'])) {
        echo "<p>Username: " . $_POST['txtName'] . "</p>";
        echo "<p>Hidden value: " . $_POST['txtHidden'] . "</p>";
    }
    else {
        echo "No values submitted";
    }
?>

Some useful PHP functions

  • isset($var) Returns a TRUE value if the variable has been set and is not a NULL value.
  • empty($var) Returns a TRUE value if the variable hasn’t been set, contains a NULL value, or contains an empty string.
  • is_numeric($var) Returns a TRUE value if the variable is a number or a string that can be converted to a number.

HTML 5 Self Validation

Self validation

  • Most of the new input types are self validating, meaning the user input will be validated on the client-side.
    • If the user input is valid, the form data is submitted to the server.
    • If the user input is invalid, a warning message will be generated to inform the user.

Self validation

  • A lot of users also have JavaScript disabled out of security reasons and hereby miss out on client-side validation.
  • ! Do take note that data should still be validated on server-side !
  • Validation can be bypassed by adding the formnovalidate attribute to the submit input control.

<input type="submit" value="Submit" formnovalidate="formnovalidate">

HTML5 form input attributes

Autofocus

  • "autofocus"
    • Selects the input control that automatically receives focus after the page is loaded. This allows the user to immediately start typing without first having to select the input field.
    • Can only be used on one input control on a form.

<input type="url" name="website" autofocus="autofocus">

Required

  • "required"
    • The required attribute indicates which input control fields must contain a valid value before the form can be submitted.
    • These fields can not be empty.
    • Can be used with any input control.

<input type="date" name="dateofevent" required="required">

Autocomplete

  • "required"
    • The autocomplete attribute allows users to select values for input types based on previous information provided by the user.
    • Can be enabled for the entire form or can be enabled on specific controls.
    • Should be disabled for credit card information and password fields.
    • Autocomplete only works when you give the control a name and/or id.

<input type="text" name="address" required="required">
